## 1. Постановка

Универсальный метод формализации произвольного объекта — гипотезы, процесса, плана,
схемы, функции — в структурную **модель ветвления**: орграф с типизированными рёбрами.

Точный термин — не «оцифровка», а **структурная формализация** (извлечение модели):
гипотезы и планы уже символьны. Конвейер: репрезентация (объект → элементы +
отношения) → структура → диагностика. Универсальность достигается тем, что различия
классов объектов выталкиваются в **типизацию рёбер**: конвейер один, меняется словарь
отношений. Универсальная промежуточная форма — триада «элементы + типизированные
отношения + порядок»; причинность, темпоральность, иерархия — не разные структуры,
а разные метки рёбер одного типизированного орграфа.

Фракталы, степенной закон, симметрия — не методы генерации модели, а **свойства** уже
построенной модели, каждое со строгим тестом: циклы — через конденсацию сильно
связных компонент; симметрия — группа автоморфизмов; фрактальность — box-covering;
степенной закон — только строгим статистическим тестом (Clauset–Shalizi–Newman),
не визуальной подгонкой.

## 2. Томографическая постановка: реконструкция мастер-объекта

Направление конвейера — не индукция «данные → модель», а **реконструкция**:
существует мастер-объект M, а наблюдаемые ветвления — его срезы-проекции π₁(M),
π₂(M)… Задача обратная, как в томографии.

Следствия. Верификация — это проверка, склеиваются ли проекции как виды одного M
(пучковая склейка; топологический анализ данных занимает здесь правильное место — не
извлечение, а склейка). Инварианты — свойства M: цикл может проявиться только при
склейке. M принципиально недостижим полностью: рабочая модель = приближение M
с точностью до ядра проекций + **явные слепые зоны**.

## 3. Стратификация срезов

Пять срезов M, изоморфных известному разложению: (1) статика ветвления — топология;
(2) внутренняя динамика — потоки НА графе; (3) внешние потоки — граница открытой
системы; (4) история эволюции — динамика САМОГО графа; (5) старшая рабочая модель —
M как многослойная темпоральная сеть.

Срезы неравноранговы: динамика не существует без статики — диаграмма слоиста, статика
является базой. Граница «внутреннее/внешнее» — модельный выбор, не данность: явный
параметр, тестируемый агентом. Сжимающийся цикл — механизм кристаллизации: аттрактор
динамики застывает в структуру, откуда тест самосогласованности — скелет (1) выводим
как множество аттракторов динамики (2).

## 4. Развёртка: динамика есть статика большего графа

Любой процесс статизируется развёрткой: узел = (элемент × такт частичного порядка),
поток = ориентированное ребро между временными копиями. «Время» — любой частичный
порядок: исполнение для процесса, логическое следование для гипотезы, приоритет для
плана. Развёртка универсальна и для нефизических объектов.

Цена развёртки — бесконечность; её оплачивает симметрия. Бесконечная развёртка
становится конечной статикой, когда есть группа симметрий с конечным фактором:

- сдвиговая симметрия → **цикл** (периодика);
- масштабная симметрия → **фрактальный генератор + правило** (ренормгруппа);
- символическая динамика → **конечный граф-генератор + правило обхода** (хаос).

Итог: **динамика = конечный статический граф-генератор + правило обхода** — три
режима одного механизма. Две динамики эквивалентны, если их генераторы сопряжены
(shift equivalence). Дуализм «волна/корпускула» снимается: корпускула = узел,
волна = орбита симметрии — два фактора одной развёртки. Обязательный инвариант —
ориентация рёбер (стрела времени).

## 5. Два примитива и отсутствие внешнего

Схема сжимается до двух примитивов: **объект** и **работа** — две факторизации одной
развёртки (структура = аттракторы динамики). «Внешнего» потока не существует:
внешнее(k) ≡ внутреннее(k+1) на границе модуля — это более высокий масштаб, где
работа целого становится узлом уровня выше. Фрактальность становится архитектурой:
самоподобная башня пар (объект, работа), связанных оператором масштаба.

Критерий допустимого шага масштаба — **lumpability**: фактор динамики = динамика
фактора; некоммутирующее разбиение «режет по живому». Проверка «похоже на данные»
заменяется проверкой «этажи коммутируют». Внешнее не исчезает — вытесняется на
вершину башни (горизонт с краевыми условиями). Степенной закон получает
предсказательный статус: при самоподобии распределение размеров модулей обязано
быть степенным — это тест самоподобия башни.

## 6. ДНК-граф: универсальный структурный алфавит

**ДНК-граф — новое понятие, не биологическая ДНК.** Это универсальный алфавит
структурных примитивов, из которого композицией собираются все реальные структуры.

Дифференциация структур (условные «клетка» и «орган» проекта) — **композиционная,
не экспрессионная**: один алфавит, разные слова. Вопрос «если ДНК одна, откуда
различие органов» не возникает — различие в сборке, а не в чтении кода.

Математически алфавит — множество **неразложимых** графов относительно выбранной
операции композиции ⊗. Точный аналог механизма — voltage graphs (Gross–Tucker):
малая база + групповые метки порождают накрытие произвольного размера; ключ
хранения — каноническая форма. Условие каноничности («чертоги бога» — единое
основание, а не один базис из многих): ⊗ допускает **единственное** разложение на
простые — тогда алфавит не назначается, а **обнаруживается**, он вынужден операцией.
Для декартова произведения графов единственность доказана (Sabidussi–Vizing);
ближайший к вложенности «клетка → орган» кандидат — **модульная композиция**
(подстановка подграфа в вершину) с её единственным каноническим деревом модульного
разложения: ДНК-граф = листья-примитивы этого дерева.

Экономия вычислений (FLOPs) — из трёх источников: инварианты (спектр, группа
автоморфизмов, гомологии, аттракторы) считаются на класс, а не на экземпляр;
симметрия делит перебор в |Aut| раз; самоподобие переиспользует шаблоны по всем
масштабам. Степенной закон предсказывает, где выигрыш: тяжёлая голова немногих
шаблонов покрывает основную массу структур. Предостережение: ориентация ломает
симметрию базы (Aut ориентированного ≤ Aut неориентированного) — инварианты шаблона
являются верхними оценками с флагом «требует проверки». Размер базового графа
ограничен сверху: крупные структуры — композиция малых, а не новые крупные шаблоны
(«кодонов мало, белков много»).

## 7. Клетка: замыкание петли производства

Предел роста структурной единицы подсказан природой, и это не молекула, а **клетка**.
Молекула держится извне (энергией связей); клетка держится изнутри — поддерживает
собственную границу активной работой. Это операциональная замкнутость (автопоэзис
Maturana–Varela; в графовых терминах — (M,R)-системы Розена).

Формально: петля **ДНК → работа → ДНК** замкнута — работа порождает объект, объект
несёт работу. Эта петля закрывает регресс Розена («кто чинит ремонтёра»)
единственным возможным способом: код кодирует машину, читающую код; базовый шаблон
самореферентен. Следствия: атом базы — не просто подграф, а **неподвижная точка**
отображения работа → структура → работа (фильтр отсеивает устойчивые, но мёртвые
структуры: кристалл компактен, но себя не производит); взаимоподобие объекта и
работы перестаёт быть гипотезой — оно конститутивно для клетки. Практическая
развязка вычислительной трудности: дешёвый конструктор + строгий фильтр живого.

## 8. Три уровня: код, процесс, поле

Три уровня организации — три математических режима, а не три размера:

- **ДНК-граф** — код: порождение и хранение (комбинаторно-групповая математика);
- **клетка** — процесс: самоподдержание, неподвижная точка петли (динамические системы);
- **жизнь** — поле: взаимодействие и отбор моделей в общей базе знаний
  (экология, теория игр на сети моделей).

Переходы содержательны: ДНК → клетка = развёртка + оживление (большинство развёрток
не самоподдерживаются); клетка → жизнь = размыкание границы (внешнее клетки =
внутреннее поля). Каждый переход несёт свой критерий отбора: код разворачивается →
клетка самоподдерживается → модель выживает среди других. Метод самореферентен:
он сам есть ДНК (метод) → клетки (модели) → жизнь (поле знаний).

## 9. Верификация

Модель проверяется агентом не подгонкой к данным, а внутренней согласованностью:

1. ориентация рёбер сохранена (стрела времени);
2. этажи коммутируют — фактор динамики = динамика фактора (lumpability);
3. скелет статики выводим как множество аттракторов динамики;
4. симметрии устойчивы к bootstrap-ресемплингу (persistence);
5. распределение размеров модулей — степенное (строгий тест, не визуальный);
6. петля производства ДНК → работа → ДНК замкнута.

## 10. Открытая проблема

Самая хрупкая точка, требующая закрытия раньше любого кода, — **выбор операции
композиции ⊗**. Пока она не выбрана, каноничность единого алфавита («чертогов
бога») — не теорема, а требование. Примитивы вторичны: они следуют из операции.
