## Суть идеи

Универсальный метод формализации произвольного объекта — гипотезы, процесса,
плана, схемы, функции — в структурную **модель ветвления** (орграф с типизированными
рёбрами). Один конвейер для всех классов объектов: различия классов выносятся в
семантику рёбер, а не в архитектуру метода.

Метод строится вокруг трёх положений.

### 1. Развёртка: динамика есть статика большего графа

Любой процесс статизируется через развёртку: узел = (элемент × такт частичного
порядка), поток = ориентированное ребро между временными копиями. «Время» —
любой частичный порядок: для процесса — исполнение, для гипотезы — логическое
следование, для плана — приоритет.

Конечной статикой бесконечная развёртка становится тогда, когда у неё есть группа
симметрий, фактор по которой конечен:

- сдвиговая симметрия → **цикл** (периодика);
- масштабная симметрия → **фрактальный генератор + правило** (ренормгруппа);
- символическая динамика → **конечный граф-генератор + правило обхода** (хаос).

Итог: **динамика = конечный статический граф-генератор + правило обхода.**
Объект и работа — две факторизации одной развёртки (структура = аттракторы
динамики). «Внешнего» потока не существует: это лишь более высокий масштаб, где
работа целого становится узлом уровня выше. Отсюда — самоподобная башня пар
(объект, работа), связанных оператором масштаба.

### 2. ДНК-граф: универсальный структурный алфавит

**ДНК-граф — новое понятие, не биологическая ДНК.** Это универсальный алфавит
структурных примитивов, из которого композицией собираются все реальные структуры.
Разные структуры (условная «клетка» и «орган» проекта) различаются **композиционно**
— как разные слова из одних букв, — а не разным прочтением одного кода.

Алфавит определяется как множество **неразложимых** графов относительно выбранной
операции композиции ⊗. Если ⊗ допускает единственное разложение на простые (напр.,
модульное разложение с его каноническим деревом), алфавит не назначается, а
**обнаруживается** — он вынужден операцией и каноничен. Это и есть «чертоги бога»:
единое основание, а не один базис из многих.

Реальный орграф тогда не хранится целиком — хранится ссылка на базу + паттерн
сборки. Инварианты (спектр, группа автоморфизмов, гомологии, аттракторы) считаются
один раз на класс, а не на экземпляр. Цель — **резкое сокращение вычислительных
затрат (FLOPs)** за счёт словарного переиспользования шаблонов по всем масштабам.

### 3. Клетка и три уровня

Единица башни — **клетка**: минимальная структура, где петля производства замкнута
на собственный код (ДНК → работа → ДНК). Эта петля закрывает регресс ремонта
(«кто чинит ремонтёра») — базовый шаблон самореферентен. Три уровня организации:

- **ДНК-граф** — код (порождение, хранение);
- **клетка** — процесс (операциональная замкнутость, самоподдержание);
- **жизнь** — поле (взаимодействие и отбор моделей в общей базе знаний).

Каждый переход несёт собственный критерий отбора: разворачиваемость →
самоподдержание → выживание среди других моделей.

### Верификация

Модель проверяется агентом не подгонкой к данным, а внутренней согласованностью:
сохранение ориентации (стрелы времени), **коммутативность этажей** (фактор
динамики = динамика фактора), **устойчивость к ресемплингу** (persistence), и
**степенной закон** распределения размеров модулей как тест самоподобия башни.
